## Core Fundamentals

### 1. You are given a manufacturing defect dataset with missing values, outliers, and class imbalance. How would you structure your initial analysis before modeling? 🟡

**Answer:** I would start with data quality checks, target leakage assessment, and exploratory analysis segmented by time, plant, and product line. Then I would handle missingness based on mechanism, investigate outliers with domain context, and choose imbalance strategies such as class weights or resampling only after defining the business metric.

*Difficulty: Medium*

---

### 2. When would you prefer hypothesis testing over machine learning for a business problem? 🟡

**Answer:** Hypothesis testing is better when the goal is to validate a specific claim, estimate statistical significance, or compare groups under controlled assumptions. Machine learning is more suitable when the goal is prediction at scale and the relationship between variables may be complex or non-linear.

*Difficulty: Medium*

---

### 3. A model shows 98% accuracy in defect prediction, but operations says it is not useful. What would you check first? 🟢

**Answer:** I would check class imbalance and whether accuracy is masking poor recall on the defect class. Then I would review precision, recall, PR-AUC, confusion matrix, threshold selection, and whether the evaluation setup matches the real production distribution.

*Difficulty: Easy*

---

### 4. Explain bias-variance tradeoff in the context of forecasting sales across regions and brands. 🟡

**Answer:** High-bias models may underfit and miss local seasonality or brand-specific effects, while high-variance models may overfit noise in sparse regional data. The goal is to choose a model and feature set that generalizes well across time and hierarchy levels, often validated through rolling forecasts.

*Difficulty: Medium*

---

### 5. How would you detect and prevent data leakage in a time-series forecasting pipeline? 🔴

**Answer:** I would ensure all features are created using only information available up to the prediction timestamp, including lag, rolling, and external variables. Validation should be time-based rather than random, and I would audit feature generation code to catch future-derived aggregates or target-informed imputations.

*Difficulty: Hard*

## Python

### 1. How would you optimize a slow Python data pipeline that processes millions of records daily? 🟡

**Answer:** I would first profile the code to identify bottlenecks, then reduce Python-level loops by using vectorized pandas or NumPy operations where possible. If memory or scale is the issue, I would move heavy transformations to PySpark, chunk processing, or parallelize carefully for CPU-bound tasks.

*Difficulty: Medium*

---

### 2. What are the tradeoffs between using pandas and PySpark in a production ML pipeline? 🟡

**Answer:** Pandas is simpler and faster for small to medium in-memory datasets, while PySpark is better for distributed processing and large-scale ETL. The tradeoff is that PySpark adds cluster overhead and more complex debugging, so the choice depends on data volume, latency, and operational constraints.

*Difficulty: Medium*

---

### 3. How would you design reusable Python code for feature engineering across multiple forecasting models? 🟡

**Answer:** I would separate feature generation into modular, parameterized functions or classes with clear input-output contracts and versioned configurations. This makes it easier to test, reuse across models, and ensure consistency between training and inference pipelines.

*Difficulty: Medium*

---

### 4. In Python, what is the difference between shallow copy and deep copy, and why does it matter in ML workflows? 🔴

**Answer:** A shallow copy copies the outer container but keeps references to nested objects, while a deep copy recursively copies all nested objects. This matters when modifying feature dictionaries, model configs, or nested metadata, because unintended shared references can create hard-to-debug training inconsistencies.

*Difficulty: Hard*

---

### 5. You need to expose a model inference service in Python for OCR-based document extraction. What engineering concerns would you address? 🔴

**Answer:** I would address request validation, batching, latency, concurrency, model loading strategy, and error handling for corrupt or large files. I would also include logging, monitoring, versioning, and fallback behavior because OCR pipelines often fail on edge-case document formats.

*Difficulty: Hard*

## Machine Learning

### 1. How would you choose between Random Forest, AdaBoost, and XGBoost for a tabular defect prediction problem? 🟡

**Answer:** I would compare them based on data size, noise sensitivity, interpretability needs, and latency constraints using cross-validated business metrics. Random Forest is robust and easy to baseline, boosting methods often perform better on structured data, and XGBoost usually offers stronger control over regularization and missing values.

*Difficulty: Medium*

---

### 2. Your segmentation model performs well offline but poorly on new plant images. What are the most likely causes? 🔴

**Answer:** The most likely causes are domain shift, annotation inconsistency, different camera conditions, or preprocessing mismatch between training and production. I would inspect failure slices, compare data distributions, and validate whether the model is overfitting to plant-specific backgrounds or lighting artifacts.

*Difficulty: Hard*

---

### 3. How would you evaluate a recommendation or text classification system that routes process non-conformities into categories? 🟡

**Answer:** I would use class-wise precision, recall, F1, and confusion analysis because some categories are likely more costly to misclassify than others. If the system is used for ranking candidate categories, I would also evaluate top-k accuracy and collect human feedback on ambiguous cases.

*Difficulty: Medium*

---

### 4. What is the practical difference between bagging and boosting? 🟢

**Answer:** Bagging trains models independently and reduces variance by averaging, while boosting trains sequentially and focuses on correcting previous errors to reduce bias. In practice, bagging is often more stable on noisy data, while boosting can achieve higher accuracy but may overfit if not regularized.

*Difficulty: Easy*

---

### 5. How would you build a robust feature selection strategy for multivariate sales forecasting without overfitting? 🔴

**Answer:** I would combine domain-driven candidate generation with time-aware validation, correlation and leakage checks, and model-based importance measured only on training folds. The key is to evaluate feature stability across rolling windows rather than selecting features from the full dataset once.

*Difficulty: Hard*

## Computer Vision & NLP

### 1. For extracting text and relationships from engineering diagrams in PDFs, how would you design the pipeline end to end? 🔴

**Answer:** I would separate the problem into document parsing, image preprocessing, OCR, object detection or segmentation for symbols and connectors, and graph or rule-based post-processing to infer relationships. The final stage would normalize outputs into a structured schema with confidence scores and human review hooks for low-confidence cases.

*Difficulty: Hard*

---

### 2. When would you choose UNet over Faster R-CNN, and when would you combine them? 🟡

**Answer:** UNet is better for pixel-level segmentation tasks such as liquid level boundaries, while Faster R-CNN is better for detecting object instances with bounding boxes. I would combine them when I need both localization and precise shape information, for example detecting a gauge region first and then segmenting the relevant area.

*Difficulty: Medium*

---

### 3. How would you improve OCR accuracy on noisy industrial images or analog gauges? 🟡

**Answer:** I would improve image quality through denoising, contrast enhancement, perspective correction, and region-of-interest detection before OCR. I would also fine-tune OCR or use domain-specific post-processing such as expected character patterns, unit constraints, and temporal smoothing across frames.

*Difficulty: Medium*

---

### 4. A RAG system over legal documents returns fluent but incorrect answers. How would you troubleshoot it? 🔴

**Answer:** I would separately evaluate retrieval quality, chunking strategy, metadata filtering, and generation grounding to determine whether the issue is retrieval failure or hallucination. Common fixes include better chunk boundaries, hybrid search, reranking, stricter prompting, and requiring citation-backed answers from retrieved context only.

*Difficulty: Hard*

---

### 5. What are the key differences between lexical, syntactic, and semantic NLP techniques in a real classification system? 🔴

**Answer:** Lexical methods focus on words and frequency patterns, syntactic methods use grammatical structure, and semantic methods capture meaning and context. In practice, lexical features are fast baselines, syntactic features help in structured language, and semantic embeddings usually perform best when wording varies across similar issues.

*Difficulty: Hard*

**Answer:** I would separate ingestion, validation, feature generation, training, evaluation, and inference into orchestrated jobs with environment-controlled configs. I would use MLflow for experiment tracking and model registry, Delta tables for reliable data handling, and automated tests plus monitoring for drift and failures.

### 2. What would you log and monitor for a computer vision model deployed in production? 🟡

**Answer:** I would monitor input data quality, prediction confidence, latency, throughput, failure rates, and business outcomes such as defect catch rate or false alarm rate. For vision specifically, I would also track image property drift like brightness, blur, resolution, and camera source changes.

*Difficulty: Medium*

### 3. How do you ensure reproducibility in an ML pipeline across development and production environments? 🟡

**Answer:** I would version code, data snapshots, feature logic, model artifacts, and dependencies, and store training parameters with each run. Reproducibility also requires deterministic preprocessing where possible and a clear lineage from raw data to deployed model version.

*Difficulty: Medium*

---

### 4. A retrained model performs better offline but worse after deployment. How would you investigate? 🔴

**Answer:** I would compare offline and online feature generation, serving-time transformations, threshold settings, and traffic segmentation to detect train-serve skew. Then I would review shadow predictions, calibration, and whether the offline validation set truly represented current production conditions.

*Difficulty: Hard*

---

### 5. What deployment strategy would you use for a high-impact industrial inspection model, and why? 🔴

**Answer:** I would use shadow or canary deployment first to compare predictions safely against the current system before full rollout. This reduces operational risk, allows targeted rollback, and gives time to validate performance on real edge cases that are often missing in offline test sets.

*Difficulty: Hard*

## System Design & Architecture

### 1. Design a scalable architecture for a RAG application that answers questions from company legal documents. 🔴

**Answer:** I would use a pipeline with document ingestion, OCR if needed, chunking, embedding generation, vector storage, metadata indexing, retrieval, reranking, and an LLM answer service with citations. The design should include access control, document versioning, caching, observability, and fallback behavior when retrieval confidence is low.

*Difficulty: Hard*

---

### 2. How would you architect a real-time analog gauge reading system for industrial robots? 🔴

**Answer:** I would split it into edge-side image capture and preprocessing, a low-latency inference service for gauge detection and OCR, and a downstream calibration and alerting service. The system should tolerate intermittent connectivity, support model updates, and include confidence-based escalation for uncertain readings.

*Difficulty: Hard*

### 3. What are the main design considerations for a batch versus streaming ML system? 🟡

**Answer:** Batch systems optimize throughput and cost for periodic processing, while streaming systems optimize latency and continuous updates. The choice depends on business response time, data arrival patterns, feature freshness requirements, and operational complexity.

*Difficulty: Medium*

---

**Answer:** I would use a dimensional model with fact tables for production, defects, and inspections, and dimensions for plant, line, product, time, and defect taxonomy. This supports consistent KPI definitions, drill-down analysis, and efficient aggregation in BI tools like Tableau or Power BI.

*Difficulty: Medium*

---

**Answer:** I would enforce versioned datasets, feature definitions, and model artifacts with lineage captured in a central registry. Clear promotion workflows, immutable artifacts, and environment-specific approvals help prevent teams from deploying incompatible or untraceable models.

## Problem Solving & Algorithms

### 1. How would you detect whether two engineering diagram components are connected when OCR text and line detections are noisy? 🔴

**Answer:** I would model the diagram as a graph where nodes are detected components or text anchors and edges are candidate connections derived from line geometry and proximity rules. Then I would use confidence scoring and graph cleanup heuristics to resolve ambiguous links and validate against domain constraints.

*Difficulty: Hard*

---

### 2. What is the time complexity of searching for an item in a balanced binary search tree versus a hash table, and how does that affect system design? 🟡

**Answer:** A balanced binary search tree gives O(log n) search, while a hash table is O(1) average-case but can degrade in worst cases. In system design, hash tables are preferred for fast lookups, while trees are useful when ordered traversal or range queries are required.

*Difficulty: Medium*

---

### 3. You need to process a large stream of sensor readings and continuously report the top-k highest temperatures. What approach would you use? 🟡

**Answer:** I would maintain a min-heap of size k so each new reading can be compared against the smallest current top value efficiently. This gives O(log k) update time and is practical for streaming scenarios where storing all readings is unnecessary.

*Difficulty: Medium*

---

### 4. How would you design an algorithm to smooth noisy liquid level predictions over time without hiding real drops? 🔴

**Answer:** I would combine temporal filtering with change-point or rate-of-change logic so short-term noise is suppressed but sustained drops are preserved. In practice, exponential smoothing or a Kalman-style approach with alert thresholds often works better than naive moving averages alone.

*Difficulty: Hard*

---

### 5. Explain dynamic programming through a practical example relevant to analytics or NLP. 🔴

**Answer:** Dynamic programming solves problems by storing solutions to overlapping subproblems, such as edit distance between two text strings. In NLP, this is useful for fuzzy matching or OCR correction because it efficiently computes the minimum number of insertions, deletions, and substitutions needed to transform one string into another.

*Difficulty: Hard*
