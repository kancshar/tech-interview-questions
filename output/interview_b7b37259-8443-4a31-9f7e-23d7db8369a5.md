## Core Fundamentals

### 1. What is the difference between monolithic and microservices architecture? 🟡

**Answer:** Monolithic architecture is a self-contained system with all components built together, while microservices architecture is a collection of small, independent services that communicate with each other. Microservices provide more flexibility and scalability. This approach is useful for large, complex systems.

*Difficulty: Medium*

---

### 2. How do you handle errors in a distributed system? 🟡

**Answer:** Error handling in a distributed system involves implementing retries, timeouts, and circuit breakers to prevent cascading failures. It's also essential to log and monitor errors to identify and fix issues quickly. This ensures the system remains stable and functional.

*Difficulty: Medium*

---

### 3. What is the trade-off between consistency and availability in a distributed system? 🔴

**Answer:** In a distributed system, consistency ensures that all nodes have the same data, while availability ensures that the system is always accessible. The trade-off is that prioritizing consistency can reduce availability, and vice versa. The CAP theorem states that it's impossible to achieve all three simultaneously.

*Difficulty: Hard*

---

### 4. What is the purpose of a load balancer in a system? 🟢

**Answer:** A load balancer distributes incoming traffic across multiple servers to improve responsiveness, reliability, and scalability. It helps to prevent any single server from becoming overwhelmed and becoming a bottleneck, ensuring that the system remains stable under heavy loads.

*Difficulty: Easy*

## Machine Learning

### 1. What is overfitting in machine learning, and how do you prevent it? 🟡

**Answer:** Overfitting occurs when a model is too complex and performs well on training data but poorly on new, unseen data. To prevent overfitting, techniques such as regularization, early stopping, and cross-validation can be used to reduce model complexity and improve generalization.

*Difficulty: Medium*

---

### 2. What is the difference between supervised and unsupervised learning? 🟢

**Answer:** Supervised learning involves training a model on labeled data to make predictions, while unsupervised learning involves training a model on unlabeled data to discover patterns or relationships. Supervised learning is used for tasks like classification and regression, while unsupervised learning is used for tasks like clustering and dimensionality reduction.

*Difficulty: Easy*

---

### 3. How do you evaluate the performance of a machine learning model? 🟡

**Answer:** The performance of a machine learning model is evaluated using metrics such as accuracy, precision, recall, F1 score, mean squared error, and R-squared. The choice of metric depends on the specific problem and the type of model used. Cross-validation is also used to assess the model's ability to generalize to new data.

*Difficulty: Medium*

---

### 4. What is the concept of bias-variance trade-off in machine learning? 🔴

**Answer:** The bias-variance trade-off refers to the balance between a model's ability to fit the training data (bias) and its ability to generalize to new data (variance). A model with high bias pays little attention to the training data and oversimplifies, while a model with high variance is too complex and overfits. The goal is to find a balance between the two.

*Difficulty: Hard*

## RAG

### 1. What is RAG, and how does it relate to machine learning? 🟡

**Answer:** RAG (Retrieve, Augment, Generate) is a framework for building machine learning models that involve retrieving relevant information, augmenting it with additional data or context, and generating output based on the retrieved and augmented data. RAG is particularly useful for natural language processing tasks.

*Difficulty: Medium*

---

### 2. How does RAG improve the performance of machine learning models? 🟡

**Answer:** RAG improves the performance of machine learning models by providing them with relevant and contextual information, which enables the models to make more accurate predictions and generate more coherent output. RAG also helps to reduce the need for large amounts of labeled training data.

*Difficulty: Medium*

---

### 3. What are the key components of a RAG system? 🟢

**Answer:** The key components of a RAG system include a retriever, an augmenter, and a generator. The retriever is responsible for retrieving relevant information, the augmenter is responsible for augmenting the retrieved information with additional data or context, and the generator is responsible for generating output based on the retrieved and augmented data.

*Difficulty: Easy*

---

### 4. How do you evaluate the performance of a RAG system? 🔴

**Answer:** The performance of a RAG system is evaluated using metrics such as accuracy, F1 score, and ROUGE score, which measure the quality of the generated output. The system's ability to retrieve relevant information and augment it with additional data or context is also evaluated using metrics such as recall and precision.

## Problem Solving & Algorithms

### 1. How do you solve the problem of finding the maximum subarray sum in an array? 🟡

**Answer:** The maximum subarray sum problem can be solved using Kadane's algorithm, which iterates through the array and keeps track of the maximum sum of a subarray ending at each position. The algorithm has a time complexity of O(n) and a space complexity of O(1).

*Difficulty: Medium*

---

### 2. What is the time complexity of the binary search algorithm? 🟢

**Answer:** The time complexity of the binary search algorithm is O(log n), where n is the number of elements in the array. This is because the algorithm divides the search space in half at each step, reducing the number of comparisons needed to find the target element.

*Difficulty: Easy*

---

### 3. How do you solve the problem of finding the shortest path in a graph? 🔴

**Answer:** The shortest path problem can be solved using Dijkstra's algorithm or Bellman-Ford algorithm, depending on the type of graph and the presence of negative weight edges. Dijkstra's algorithm has a time complexity of O((V+E)logV) and is suitable for graphs with non-negative weight edges.

*Difficulty: Hard*

---

### 4. What is the purpose of the sliding window technique in algorithm design? 🟡

**Answer:** The sliding window technique is used to solve problems that involve finding a subset of data that meets certain conditions, such as finding the maximum sum of a subarray or the longest substring without repeating characters. The technique involves maintaining a window of elements and sliding it over the data to find the desired subset.

*Difficulty: Medium*
